# Техническое задание
## Проект: тестирование веб-приложения Trello

## 1. Общая информация о проекте
**Trello** — это веб-приложение для организации личных и командных задач по принципу канбан-досок.  
Пользователь может создавать доски, списки и карточки, распределять задачи по этапам выполнения, назначать исполнителей, сроки и отслеживать статус работ.

**Тип проекта:** веб-приложение.

## 2. Цель сайта
Цель приложения Trello — упростить планирование, распределение и контроль задач как для индивидуальных пользователей, так и для команд.

## 3. Задачи сайта
Основные задачи приложения:
- создание и хранение задач в удобной визуальной форме;
- распределение задач по этапам выполнения;
- организация совместной работы команды;
- контроль сроков выполнения задач;
- повышение прозрачности рабочего процесса;
- сокращение количества пропущенных и забытых задач.

## 4. Целевая аудитория
Целевой аудиторией приложения являются:
- студенты и учебные команды;
- небольшие и средние рабочие коллективы;
- project-менеджеры;
- фрилансеры;
- пользователи, которым необходимо планировать личные дела и проекты.

## 5. Проблемы, которые решает приложение
Trello решает следующие проблемы:
- отсутствие единого места для хранения задач;
- сложность отслеживания текущего статуса выполнения задач;
- плохая координация между участниками команды;
- риск пропуска дедлайнов;
- неудобство текстового или устного распределения задач без наглядной структуры.

## 6. Аналоги и сравнение с ними

### 6.1 Jira
**Плюсы по сравнению с Trello:**
- более мощные инструменты для крупных команд;
- развитая система отчетности;
- подходит для сложных Agile/Scrum-проектов.

**Минусы по сравнению с Trello:**
- более сложный интерфейс;
- выше порог входа для новых пользователей;
- избыточность для небольших команд и простых проектов.

### 6.2 Asana
**Плюсы:**
- удобное управление проектами и сроками;
- расширенные представления задач;
- хорошие инструменты командного взаимодействия.

**Минусы:**
- часть полезных функций ограничена платными тарифами;
- для простых сценариев может быть сложнее Trello.

### 6.3 Notion
**Плюсы:**
- гибкость настройки;
- можно совмещать задачи, заметки, базы знаний и документы;
- универсальность.

**Минусы:**
- менее специализирован как таск-трекер;
- настройка рабочего пространства требует больше времени.

### 6.4 Вывод по сравнению
Trello выигрывает за счет простоты, наглядности и быстрого освоения.  
Он особенно удобен для небольших команд, учебных проектов и пользователей, которым нужен понятный визуальный инструмент управления задачами.

## 7. Описание работы приложения
Приложение работает через веб-интерфейс в браузере.

Основной принцип работы:
1. Пользователь регистрируется или входит в систему.
2. Создает рабочее пространство и доску.
3. Добавляет списки, например: «Нужно сделать», «В работе», «Готово».
4. Создает карточки задач внутри списков.
5. Перемещает карточки между списками по мере изменения статуса.
6. Может назначать участников, сроки, чек-листы, комментарии и вложения.
7. Отслеживает текущее состояние проекта на доске.

## 8. Объект тестирования
Объектом тестирования является веб-приложение Trello.

Предполагается проверка следующих ключевых функций:
- регистрация и авторизация;
- создание и редактирование досок;
- создание, редактирование и удаление карточек;
- перемещение карточек между списками;
- добавление сроков, участников и комментариев;
- работа интерфейса и удобство основных пользовательских сценариев.
